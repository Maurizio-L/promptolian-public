PTR-Guided Adaptive Mode
Auto-selects the right compression depth
PTR (Protected Token Ratio) = protected_facts / total_tokens. When PTR > 0.65, the prompt is fact-dense — applying telegraphic grammar stripping would risk breaking meaning. Promptolian stays in standard mode. When PTR ≤ 0.65, the prompt has enough "filler" for aggressive stripping and telegraphic mode is applied automatically. Experiment result: v4 lean pass reaches ~35% avg CR vs 17.3% with v3 telegraphic — adding phrase simplification, word-level pruning, and smart abbreviation on top of symbol substitution.
Features
Everything built in
⚡
One-click compression
Click Compress in the browser toolbar. Promptolian encodes your prompt and injects it. One click to restore. Zero friction.
🔒
Fact preservation (UUID guard)
Before any rule runs, a pre-pass replaces facts (numbers, URLs, paths, versions, tech names) with UUID tokens. They are restored exactly after compression. 98.4% FPR measured across 100 prompts.
🌐
5 AI platforms
Chrome and Firefox extensions inject the toolbar on Claude.ai, ChatGPT, Gemini, Microsoft Copilot, and Perplexity. SPA-aware with MutationObserver re-injection.
📊
Session analytics
Pro: tokens saved over time, per-site breakdown, 7-day chart in the popup. Session history capped at 500 entries, stored locally.
🔧
Custom vocabulary rules
Pro: define your own phrase → abbreviation rules. Company jargon, internal project names, domain-specific terms. Applied before built-in rules.
↩
Auto-compress on send
Pro: intercepts Enter key (not Shift+Enter), compresses silently, sends. Synchronous quota check from cache ensures no delay.
✂️
Lean pass — word-level pruning
Articles (a, an, the), auxiliary verbs (is, are, was), intensifiers (very, extremely, quite), and weak pronouns are removed contextually. Negation, passive voice, and critical prepositions are always preserved.
⚡
Smart abbreviation
For long prompts (>30 words), words over 8 characters are shortened using a slang-style lookup table (configuration → cfg, initialization → init, parameters → params) plus suffix-drop rules for unknown long words.
🔑
100% local — no tracking
Standard compression runs entirely in your browser. Your prompts never leave your machine. No telemetry, no logs.
🔌
REST API
Developer tier: POST /v1/compress with engine, domain, and language params. Batch endpoint. Python & JS SDK.
📈
Research dashboard
Localhost Flask dashboard with live compression comparison (Standard vs NLP vs MLM), Groq LLM quality scoring, and domain experiment visualisations.

Deploy to production

Four steps to take Promptolian from localhost to a real product.

Step 1

Deploy API + Dashboard → Railway

Connect your GitHub repo to Railway. Add a PostgreSQL plugin (free tier: 500 MB). Set env vars: DATABASE_URL, STRIPE_SECRET_KEY, STRIPE_WEBHOOK_SECRET, GROQ_API_KEY. Railway auto-deploys on push from public/. Cost: ~$5/mo for starter.

Step 2

Host website → Vercel or Cloudflare Pages

Drag-and-drop public/website/ to Vercel, or push to GitHub and connect Cloudflare Pages. Both are free. Buy domain at Namecheap or Cloudflare Registrar (~$10/yr for .com). Point DNS to Vercel/Cloudflare.

Step 3

Publish extensions → Chrome & Firefox stores

Chrome Web Store: $5 one-time dev fee. Zip public/extension/chrome/, upload, fill store listing. Review takes 1–3 days. Firefox Add-ons: free, zip public/extension/firefox/, submit. Both require a privacy policy URL.

Step 4

Set up payments → Stripe

Create Stripe account. Add STRIPE_SECRET_KEY + STRIPE_WEBHOOK_SECRET to Railway. Create two Price objects (Pro $7/mo, Dev $19/mo). Update api_server.py with the Price IDs. Test with Stripe CLI: stripe listen --forward-to localhost:3001/webhook.
